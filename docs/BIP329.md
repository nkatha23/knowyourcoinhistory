# BIP-329 Label Format

Reference: https://github.com/bitcoin/bips/blob/master/bip-0329.mediawiki

BIP-329 defines a standard JSON Lines (`.jsonl`) format for attaching human-readable labels to Bitcoin wallet entities. KYCC uses this format for import and export so labels are portable across compatible wallets.

---

## Format

Each line in the file is a JSON object:

```jsonl
{"type": "tx",   "ref": "f4184fc596403b9d...", "label": "First BTC payment"}
{"type": "utxo", "ref": "f4184fc596403b9d...:0", "label": "Hal Finney output", "spendable": true}
{"type": "addr", "ref": "bc1qar0srrr7xfkvy5l643lydnw9re59gtzzwf5mdq", "label": "Cold storage"}
{"type": "xpub", "ref": "xpub6CUGRUo...", "label": "Ledger main"}
```

| Field | Values | Notes |
|-------|--------|-------|
| `type` | `tx` `utxo` `addr` `xpub` | Required |
| `ref` | txid / `txid:vout` / address / xpub | Required |
| `label` | any string | Required, non-empty |
| `spendable` | `true` / `false` | Optional, only meaningful for `utxo` |
| `origin` | derivation path string | Optional, ignored on import |

---

## KYCC Extensions

KYCC adds `wallet_id` as an internal field during storage. This field is **not** written to the exported `.jsonl` — the file stays spec-compliant. On import, labels are assigned to the wallet ID specified in the query parameter.

---

## Export

```bash
curl "http://127.0.0.1:5050/api/labels/export?wallet_id=default" -o labels.jsonl
```

From the UI: click the **Download** icon in the toolbar. The file is saved as `kycc-labels-{wallet_id}-{YYYYMMDD}.jsonl`.

---

## Import

```bash
curl -X POST "http://127.0.0.1:5050/api/labels/import?wallet_id=default" \
     -F "file=@labels.jsonl"
# {"ok": true, "imported": 42, "wallet_id": "default"}
```

From the UI: click the **Upload** icon in the toolbar and select a `.jsonl` or `.json` file. Existing labels with the same `(wallet_id, ref_type, ref)` primary key are overwritten (upsert).

## Compatibility

| Wallet | Import | Export |
|--------|--------|--------|
| Sparrow Wallet | Yes | Yes |
| Electrum 4.x | Yes | Yes |
| Bitcoin Core 27+ | Yes | Yes |
| KYCC | Yes | Yes |

## Multi-wallet Isolation

Labels are namespaced by `wallet_id`. Two wallets can label the same UTXO differently without conflict:

```jsonl
# wallet_id = "hot"
{"type": "utxo", "ref": "abc123:0", "label": "Change from exchange"}

# wallet_id = "cold"
{"type": "utxo", "ref": "abc123:0", "label": "Received from hot wallet"}
```

Switch wallets using the selector in the toolbar. The graph reloads labels for the selected wallet on each node fetch.
